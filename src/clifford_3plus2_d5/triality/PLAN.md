# Triality Sidecar — Plan

Status: scaffolding complete, kill-test plan defined, not yet implemented.

## The load-bearing question

> Can explicit Spin(8) triality produce three equivalent SM-generation
> carriers without declaring three generations by hand?

This is binary.  If yes, the sidecar grows into a mass/mixing/CP
investigation.  If no, the sidecar is closed with a published negative
result and no mass-fitting work is done.

## The structural tension surfaced by reading lepton

Reading `lepton/clifford_octonion.py`, `lepton/clifford_patisalam.py`,
`lepton/patisalam_sm.py`, and `lepton/sm_hypercharge.py` exposes the
genuine obstruction the kill test must confront.

**Spin(8) has Z/3 outer-automorphism triality.**  Spin(10) does not — its
outer automorphism group is only Z/2.  So if the program wants Z/3
triality acting on three generations, the action must live inside Spin(8),
which is a proper subgroup of Spin(10).

**Pati-Salam has rank 5**: `SU(4) x SU(2)_L x SU(2)_R` with Cartan ranks
`3 + 1 + 1`.  **Spin(8) has rank 4.**  So at least one rank-1 SM Cartan
direction must live outside any Spin(8) ⊂ Spin(10).  This is not a soft
obstruction; it is structural.

The natural embedding `Spin(8) ⊂ Spin(10)` using Cl(0,10) gamma indices
`0..7` has:

- the full `Spin(6) = SU(4) ⊃ SU(3)_c x U(1)_{B-L}` inside (`spin06`
  generators are `combinations(range(6), 2)`, all using indices < 8);
- a single Cl(0,4) bivector `γ_6 γ_7` inside (one rank-1 piece of the
  rank-2 Spin(4));
- the bivectors `γ_8 γ_9`, `γ_6 γ_8`, `γ_6 γ_9`, `γ_7 γ_8`, `γ_7 γ_9`
  OUTSIDE.

Looking at the existing `su2_l_generators_from_spin04` in
`clifford_patisalam.py:271-288`, every `SU(2)_L` generator is a sum of one
"inside" bivector (`m_01 = γ_6 γ_7 / 2` factor) and one "outside"
bivector (`m_23 = γ_8 γ_9 / 2` factor, etc.).  The full `SU(2)_L`
algebra therefore **cannot** sit inside this Spin(8).

The same is true of `SU(2)_R`.

This is the load-bearing obstruction.  The kill test must either:

- **(K1a) preserve full SM under triality** — only possible if `SU(2)_L`
  fits inside Spin(8), which it does not under any of the standard
  embeddings.  Almost certainly fails.
- **(K1b) preserve a SM subgroup under triality** — possible.  The
  candidate subgroup inside `Spin(8) ⊂ Spin(10)` (indices `0..7`) is
  `SU(3)_c x U(1)_Y'` where `Y'` is the part of hypercharge built from
  generators inside Spin(8).

So **the honest kill test is K1b**: does triality preserve `SU(3)_c x
U(1)_Y'` inside Spin(8), and do the three triality copies decompose under
this subgroup as three equivalent multisets?

If this fails, triality is wrong even at the partial-SM level and the
program dies.

If it passes, the sidecar still does not yet have full SM-from-triality
— `SU(2)_L` is missing.  But it earns the right to ask the follow-on
question:

> Where does `SU(2)_L` come from in a triality-respecting way?  Does it
> emerge from triality breaking, or does it require structure outside
> Spin(8)?

That is **post-kill-test** work and is not part of this plan.

## The pinned Spin(8) embedding

For the kill test, Spin(8) is defined as the bivector subalgebra of
Cl(0,10) generated by gamma indices `{0, 1, 2, 3, 4, 5, 6, 7}`, acting on
the real-32 chiral-16 carrier via `patisalam_chiral16_block_matrix`.

This embedding is **a choice**.  It is the natural one because it includes
all of Cl(0,6) (preserving SU(4) and therefore SU(3)_c and B-L), but other
embeddings exist:

- `{0, 1, 2, 3, 4, 5, 8, 9}` — also keeps Cl(0,6), uses the other Spin(2)
  factor of Cl(0,4).
- `{0, 1, 2, 3, 6, 7, 8, 9}` — keeps Cl(0,4) intact, loses some of Cl(0,6).
- Mixed embeddings — many possibilities.

The pinned choice is `{0..7}`.  Logged in `parameter_ledger.md` as a
discrete embedding choice.  If the kill test fails on this embedding, a
second pass might try `{0..5, 8, 9}` for symmetry, but failing both kills
the program.

## The three 8-dimensional representations

Inside this Spin(8), there are three 8-dimensional irreducible
representations:

- **8_v** — the defining vector representation.  Spin(8) acts on the
  real-8 vector space spanned by gammas `γ_0, ..., γ_7` via the adjoint
  action `γ_i → [B, γ_i]` for B ∈ Spin(8) bivectors.

- **8_s** — the chiral spinor.  Inside the chiral-16 of Spin(0,10) (real
  dimension 32), the action of Spin(8) on the bivector subalgebra
  preserves a chirality projection.  This needs a second chirality
  projection (within the chiral-16) that distinguishes the two Spin(8)
  chiralities.  The candidate is the bivector `γ_6 γ_7` or `γ_8 γ_9` (in
  the original Cl(0,10) labels) acting inside the chiral-16.

- **8_c** — the conjugate chiral spinor.  The complementary chirality of
  the same Cl(0,8) chiral splitting.

Construction is concrete:

1. Build the Cl(0,8) volume element inside Cl(0,10) as
   `ω_8 = γ_0 γ_1 γ_2 γ_3 γ_4 γ_5 γ_6 γ_7` (using
   `patisalam_cl010_gamma_matrices()`).
2. Restrict `ω_8` to the chiral-16 with
   `patisalam_chiral16_block_matrix(ω_8)`.
3. The eigenspaces of this 32×32 matrix split the chiral-16 into two
   16-dimensional Spin(8)-invariant pieces.
4. Each of those 16-dim pieces contains an 8-dimensional irreducible of
   Spin(8) (either 8_s or 8_c, depending on chirality sign).
5. The complementary 8 in each 16-dim piece is part of the other chirality
   or of an additional structure — to be determined by explicit Spin(8)
   action computation.

`8_v` is constructed separately as the 8-dim defining representation,
which Spin(8) acts on by `γ_i → Λ^j_i γ_j` for `Λ ∈ SO(8)` via the spin
covering.

## The triality outer automorphism τ

The triality automorphism cyclically permutes `{8_v, 8_s, 8_c}`.  The
canonical construction:

- On the Spin(8) Dynkin diagram (D_4), the outer automorphism group is
  S_3 acting on the three "leaf" nodes.  Triality is the order-3 cyclic
  permutation.
- At the representation level, τ acts on Spin(8) generators by a specific
  linear map that exchanges the three 8-dim reps.
- A standard explicit formula uses octonion structure: for the natural
  identification `R^8 ≅ O`, the map
  `τ: γ → L_γ R_γ` (left-mult times right-mult by the corresponding
  octonion unit) permutes the three reps.

The triality sidecar will construct τ as a concrete matrix-level
automorphism on Spin(8) generators using either the Dynkin construction
or the octonion construction, whichever produces cleaner code given the
existing `octonion_left_multiplication` and `octonion_right_multiplication`
in `clifford_octonion.py`.

The choice of construction is a **convention choice**, logged in
`parameter_ledger.md`.  Different conventions give the same triality
class but different concrete matrices.

## The kill test, defined precisely

### K1 — Gauge subgroup stability

Define the **Spin(8)-internal SM subalgebra**:

```text
g_SM(8) = su3_c ⊕ u1_{Y'}
```

where:

- `su3_c` is the existing `su3_c_generators_from_su4()` (all 8 generators
  use Cl(0,6) bivectors, all inside Spin(8) since Cl(0,6) ⊂ Cl(0,8));
- `u1_{Y'}` is the Spin(8)-restricted part of the hypercharge generator
  `hypercharge_generator()`.  Specifically, the component of `Y = T3_R
  + (B-L)/2` that uses bivectors with both indices < 8.

`u1_{Y'}` requires explicit computation: decompose `Y` into its bivector
components and discard the ones touching indices 8 or 9.  This may
yield a generator that is no longer aligned with the physical
hypercharge — the kill test will use whatever `u1_{Y'}` actually is, and
the comparison in K2 will use Y-eigenvalues restricted accordingly.

**K1 passes if** `τ(g_SM(8)) = g_SM(8)` as a subspace.  That is, τ
applied to each generator of `g_SM(8)` produces a matrix that is a linear
combination of generators in `g_SM(8)`.

`τ` is allowed to permute generators within `su3_c` (e.g., the 8 Gell-Mann
analogues can be reordered) and to scale `u1_{Y'}` (since U(1) generators
can rescale).  It cannot map any color generator out of `su3_c`.

### K2 — Generation rep content preserved

For each of the three Spin(8) chiral-16 carriers `C_0, C_1, C_2`
(`C_0` = original, `C_1` = τ-rotated, `C_2` = τ²-rotated), decompose
under `g_SM(8)` into simultaneous eigenspaces of `su3_c` Cartan and
`u1_{Y'}`.

**K2 passes if** the three decompositions are isomorphic as
`(SU(3)_c, U(1)_{Y'})`-modules, meaning:

- same multiset of `Y'` eigenvalues;
- same multiplicities;
- same `SU(3)_c` rep at each eigenvalue (triplet, anti-triplet, or
  singlet).

K2 is the "are these three equivalent SM-restricted generation copies"
test.

### Combined verdict

| K1 | K2 | Verdict |
|---|---|---|
| pass | pass | Sidecar earns the right to grow.  Next: investigate where `SU(2)_L` comes from in a triality-respecting way. |
| pass | fail | Triality permutes generation copies inequivalently — wrong outer automorphism for generations. |
| fail | — | Triality does not preserve SM-inside-Spin(8) as a subgroup.  Program dies. |

## Failure modes named in advance

**F1 — `u1_{Y'}` is zero or trivial inside Spin(8).**
Diagnosis: hypercharge lives entirely in the `γ_8 γ_9` direction; nothing
of it survives the Spin(8) restriction.  In that case, K2 has no `U(1)`
to test and the kill test reduces to "is `su3_c` preserved under
triality".  This is still meaningful but narrower than the planned
audit.  Log explicitly.

**F2 — `τ` cannot be constructed as a closed matrix automorphism on the
chiral-16 carrier.**
Diagnosis: Spin(8) does have triality abstractly, but its concrete action
inside `patisalam_chiral16_block_matrix` may not extend to a matrix-level
operator.  In that case the kill test is undefined and the program dies
on a technicality before reaching K1.  Document fully if it happens.

**F3 — `τ` is order 3 but K1 fails.**
The expected most common failure: triality permutes Spin(8) reps cleanly
but the SM-inside-Spin(8) subgroup is not preserved.  Diagnosis: the SM
direction inside Spin(8) is special; triality doesn't see it as special;
the three triality copies put it in three different (inequivalent)
positions.  This is the natural way the program dies.

**F4 — K1 passes but K2 fails.**
The subgroup is preserved as a set but the rep content shifts under τ.
Diagnosis: triality acts on the rep content non-trivially; the three
generations would have different SM rep multiplicities, which is wrong.
Possible but less likely than F3.

**F5 — Both K1 and K2 pass.**
The desired outcome.  In this case the sidecar earns the right to grow,
and the next session asks the post-kill-test question (where `SU(2)_L`
comes from in a triality-respecting way).

## What this plan does NOT contain

These are deliberately deferred until the kill test produces a positive
verdict:

- mass hierarchy / Yukawa structure;
- Z/3 character analysis for CP phase;
- PDG numerical fit;
- breaking pattern parameterization;
- additional sessions T1..T6 from the original sketch;
- any consideration of three generations beyond verifying the
  triality copies are SM-equivalent.

The reviewer's central correction was that T4 (kill test) is the
existential gap, not T6 (PDG fit).  This plan accepts that fully.
