# Normalizer Report

Status: Phase 7 candidate-only normalizer oracle.

This report records the exact checks implemented in
[`src/clifford_3plus2_d5/search/normalizer.py`](../../src/clifford_3plus2_d5/search/normalizer.py)
and exposed by
[`scripts/normalizer_check.py`](../../scripts/normalizer_check.py).

## Purpose

The earlier phases verify that the declared carrier, `J`, `P_3/P_2`, gate
classifier, finite-depth update candidate, and spinor reconstruction are
internally consistent. Phase 7 asks a sharper question:

```text
Are J and the 3+2 split forced by rule data, or are they still convenient
coordinate choices?
```

The current answer is still conservative:

```text
forcedness_verdict: candidate_only
load_bearing_qca_bridge: false
```

## Exact Proxies

The checker computes:

- the linear centralizer of the declared rule-data operators,
- the infinitesimal real orthogonal normalizer preserving those operators,
- whether the normalizer preserves the declared `P_3/P_2` split,
- whether the normalizer also preserves the candidate `J`,
- the finite-dimensional algebra generated by switchable controls,
- rank-one and off-block addressability falsifiers.

The default rule-data proxy is deliberately weak: it contains only the declared
safe block projectors `P_3` and `P_2`. This is enough to check that the
candidate data preserve the split, but it is not evidence that microscopic QCA
rules force the split.

```bash
uv run python scripts/normalizer_check.py --check --expect-verdict candidate_only
```


Interpretation:

- The declared `P_3/P_2` data preserve the split by construction.
- The orthogonal normalizer is still too large to force the declared `J`.
- Continuous alternatives for `J` have not been excluded.
- Rank-three projector uniqueness has not been proved from microscopic data.
- The result is a useful candidate check, not a bridge proof.

## Falsifiers

The checker rejects controls that resolve individual axes or mix the two
blocks.

Commands:

```bash
uv run python scripts/normalizer_check.py --include-rank-one-color --expect-verdict falsified
uv run python scripts/normalizer_check.py --include-rank-one-weak --expect-verdict falsified
uv run python scripts/normalizer_check.py --include-off-block --expect-verdict falsified
uv run python scripts/normalizer_check.py --include-full-u5-controls --expect-verdict falsified
```

These cases set `addressability_algebra_safe: false` and
`forcedness_verdict: falsified`.

## Acceptance Gap

Phase 7 does not close the QCA bridge. A future load-bearing result would need
source-backed microscopic rule data whose normalizer forces `J`, forces the
structural rank-three/rank-two projectors, and still does not generate
rank-one controls inside either block.
