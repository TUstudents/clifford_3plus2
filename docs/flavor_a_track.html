The load-bearing claim

The A-track is a single block testing one idea: that the dimensionless Standard-Model flavor pattern is the spectral data of one boundary operator H_Q, read through four quantum-number projections — one per sector (ν, e, u, d) — via the Feshbach/Schur self-energy Σf(z) = Vf† (z − HQ)−1 Vf. If true, all sector differences are carried by the coupling V_f — which SM quantum numbers connect that sector to the boundary — not by different operators.

The track runs in three phases that can KILL cheaply and never overclaim: A2 checks the necessary conditions, A3a constructs the unification of the transfer sector, A3b is the honest "derive or count" ledger of the texture factors. Each gate tags every factor derived (machine-checked against its source) or free input, and reports the final count.

Net result. The A-track predicts flavor structure — the texture's group-theory / geometry factors and both CP phases (atan(√5), 5π/12), read off one boundary through four V_f projections — but the magnitude hierarchy rides on a free, fit depth embedding {0, 2, 6} that the closed algebraic/topological kills say has no derivation (N = 3 empirical). One remaining declared input survives: generation_depth_embedding_derived.

What the A-track establishes. The transfer invariant is genuinely shared (not three coincidences); the quark/lepton boundaries differ by exactly the color quantum number; couplings are SM quantum-number projections; one sterile chain H_Q yields both the lepton Σ and the quark transfer hierarchy as Schur complements; and the texture's structure factors plus both CP phases are derived predictions, with fewer free inputs than observables.
What it does not. It does not prove one explicit H_Q on the chiral-16 carrier reproduces all four Σ_f numerically (the sector shells live in different representation spaces), nor derive the magnitude hierarchy — the depth embedding {0,2,6} is a free input fit to the CKM hierarchy. Deriving it is a generation mechanism (N = 3 empirical).

Architecture

One package, three phases. The 12 gate/audit modules (u1–u3 + universality_audit; a31–a33 + unified_boundary_audit; b1–b3 + texture_provenance_audit) share a single external import surface, reuse.py, which re-exports only boundary_response / lepton symbols — no boundary physics is redefined.

No import cycle. reuse.py imports nothing from flavor_a_track. The one intra-track reference (A3-3 reading U3's color_split_is_quark_triplet_lepton_singlet) is an ordinary sibling import, not via reuse — so the surface stays acyclic.

Each gate's verdict is a pure, unit-testable decision function (e.g. parameter_count_verdict, free_inputs_verdict, texture_provenance_verdict) so every KILL branch is exercised by a negative-control test rather than asserted only on the happy path.
